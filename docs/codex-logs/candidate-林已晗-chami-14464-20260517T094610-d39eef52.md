## 2026-05-17 18:03:20 CST (+0800)

### 用户输入
用户在浏览器查看安排页后提出设计讨论：顶部多个标签筛选 chip 虽然有用但显得乱，询问是否有更好展示方式；当前安排分组顺序是未完成、放弃、完成，用户觉得未完成、完成、放弃可能更好；同时觉得“添加安排”的位置也有问题，要求先给修改意见。

### AI 最终输出
我的判断是：顶部这一排标签不应该默认全部展开。它确实有筛选价值，但在移动端第一屏里会抢走太多注意力，让“安排列表本身”变成第二优先级。更好的做法是把它降级成一个筛选工具：页面顶部只保留一行轻量工具条，比如“全部安排 · 3”加一个筛选按钮；点筛选后用底部抽屉展示“全部、日常、学习、工作、健康、家庭、其他”。如果用户已经选了某个标签，再在工具条里显示一个小 chip，例如“健康 ×”。这样标签能力还在，但默认页面会干净很多。卡片里保留主标签就够了，因为那里表达的是内容分类。

分组顺序我同意改成“未完成 -> 完成 -> 放弃”。这个顺序更符合心理模型：先看当前要处理的，再看已经闭环的，最后才看放下的。把“放弃”放在“完成”前面，会让页面中段出现一种负向归档感，不太符合这个模块想减轻焦虑的气质。“放弃”不是失败，而是归档到末尾更自然。

“添加安排”的位置我也觉得现在偏重。全宽按钮放在标题下面，会和筛选、列表争夺主视觉，而且用户每次进入安排页不一定都是为了新增。我的建议是：有内容时，把“添加安排”收成标题右侧一个小的加号按钮，或右上角“+”；空状态里再保留明显的“添加安排”按钮。这样老用户看列表更轻，新用户没内容时也知道怎么开始。

我倾向的下一版结构是：顶部标题“安排”右侧一个加号；下面一行“全部安排 · 数量”加筛选按钮；列表按“未完成 / 完成 / 放弃”展示；顶部不再常驻一排标签。
