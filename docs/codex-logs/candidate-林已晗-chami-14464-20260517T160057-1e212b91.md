## 2026-05-18 00:58:17 CST (+0800)

### 用户输入
用户反馈 V2.1 相似安排合并提示的 UI 结构不合理：卡片上“可能与已有安排相关”不应像普通标签/正文一样展示，合并到已有安排、仍然新建、暂不处理三个选项也不应塞在确认安排详情列表里。用户提出可在浮窗右上角放黄色感叹号和提示框，点击后出现三个选项，并讨论点击内容是否跳转到对应计划安排的微调页，要求一起评估更好的方案。

### AI 最终输出
我同意你的判断：现在这版信息架构确实不够好。它把“编辑这条新识别出来的安排”和“处理它与已有安排的关系”混在一个详情面板里了，所以看起来会像确认安排详情里突然长出了一组决策按钮；卡片上的绿色一行也容易被误解成标签或备注，不像一个需要用户处理的关系提示。

我赞成你说的“大方向”：把相似安排提示从正文里拿出来，变成卡片右上角的独立状态入口。但我会稍微反对“只有黄色感叹号”和“点击卡片内容直接跳已有安排”这两点。只有感叹号太像错误告警，也不够可发现；点击卡片主体如果改成跳已有安排，会破坏 V1 里“点击待确认卡片 = 编辑这条待确认安排”的稳定心智。

我建议改成这样：

1. 待确认卡片右上角放一个琥珀色小提示，不只是感叹号，而是“! 已有相关安排”这种短胶囊提示。
2. 点击这个提示，打开一个独立的底部小面板，里面展示已有安排摘要，并提供三个动作：`合并到已有安排`、`仍然新建`、`暂不处理`。
3. 已有安排摘要本身可以点击，进入已有安排详情/微调页。
4. 卡片主体点击仍然打开待确认安排编辑；来源行仍然回到来源对话；左滑忽略、右滑加入继续保留。
5. 确认编辑面板里移除现在那块很重的“三选一”区域，最多只保留一条轻提示，避免它抢走编辑内容的主流程。

这样会更清楚：卡片主体负责“确认这条新安排”，右上角提示负责“处理与旧安排的关系”。我建议下一步就按这个结构改，视觉上会更轻，行为上也不会打断已经建立好的 V1 操作习惯。
